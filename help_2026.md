# 이랑위키 — 2026 Holographic 리디자인 안내서 (`help_2026.md`)

> 2026 봄 단행한 **이랑위키 프런트엔드 리디자인**의 변경점, 새 HUD 디자인 토큰,
> 글을 작성/편집할 때의 실무 가이드를 한 문서에 정리합니다.

## 1. 새 디자인 컨셉 — 색다른 미래형 위키

기존 디자인은 다크 그라디언트 + 네온 카드 위주의 “관리자 대시보드” 느낌이었고,
중간 시안에서는 “위키피디아식 종이 백과사전” 톤도 시도했습니다. 최종 채택은
**Holographic / HUD 스타일**입니다.

핵심 컨셉:

- **Deep space base** — `#04070d` ~ `#0c1322`의 짙은 우주 톤
- **3 액센트 — 사이언 / 바이올렛 / 마젠타** 홀로그램 그라디언트
- **HUD 코너 브래킷** — 모든 패널의 좌상/우하 모서리에 작은 ⌐ ¬ 가 그어집니다
- **Mono numerics + wide tracking labels** — 통계나 라벨은 JetBrains Mono로 보여줍니다
- **글래스모픽 패널** — `backdrop-filter: blur` + 반투명 패널
- **글로우 효과** — 활성 탭, hover, 포커스 시 청록/자주색 빛 띠
- **시그널 펄스 도트** — 살아있는 시스템 느낌의 작은 LED

| 영역 | 변경 후 |
| --- | --- |
| 배경 | 우주 톤 + 라디얼 글로우(사이언 좌상, 바이올렛 우상, 마젠타 하단) + 미세 그리드 |
| 본문 폰트 | Pretendard (한글 본문), Space Grotesk (디스플레이), JetBrains Mono (라벨/숫자) |
| 링크 | 시안 (`#22D3EE`) + dashed 아래밑줄, hover 시 글로우 텍스트 그림자 |
| 좌측 영역 | 글로우 라인이 있는 사이드바 + 활성 항목에 시안 누/그라디언트 |
| 우측 영역 | HUD 인포박스(모니터 컨셉) + 문서 함수 패널 |
| 상단 | 홀로그램 그라디언트 라인 + 펄스 도트 + LIVE 배지 |
| 메시지 박스 | 그라디언트 배경의 mbox (info/warning/danger/success) |
| 탭 | Wikipedia식 종이 탭 → 작전 모드 탭 + 활성 표시 라인에 글로우 |

### 1.1 디자인 토큰

`globals.css`의 다음 스코프 안에서만 활성화됩니다:

```css
[data-section='wiki'] { ... }
```

`AppChrome`이 `/wiki`로 시작하는 경로에서만 `<html data-section="wiki">`를
세팅하기 때문에, **랑구팸·랑구대학 등 다른 영역에는 영향이 없습니다.**

```css
/* 베이스 톤 */
--wiki-bg-deep, --wiki-bg, --wiki-bg-2     /* 점점 밝아지는 3단계 어둠 */
--wiki-panel, --wiki-panel-strong          /* 반투명 글래스 패널 */
--wiki-chrome                               /* 헤더/사이드바 */

/* 잉크 */
--wiki-ink, --wiki-ink-soft, --wiki-ink-muted, --wiki-ink-faint

/* 룰 / 글로우 */
--wiki-rule, --wiki-rule-strong            /* 사이언 톤의 가는 라인 */
--wiki-glow-cyan, --wiki-glow-violet, --wiki-glow-magenta

/* 액센트 (홀로그램 3색) */
--wiki-cyan    /* #22D3EE */
--wiki-violet  /* #A78BFA */
--wiki-magenta /* #F472B6 */
--wiki-success /* #34D399 */
--wiki-warning /* #FBBF24 */
--wiki-danger  /* #FB7185 */

/* 의미 매핑 (주로 사용) */
--wiki-link        → --wiki-cyan
--wiki-link-hover  → #67E8F9
--wiki-accent      → --wiki-cyan
```

> 페이지 변수 `--page-background`, `--surface-background`도 내부에서 덮어씌우므로
> 위키 영역 안에서는 **다크 테마로 고정**됩니다 (다른 페이지의 라이트/다크/갤럭시 테마 토글에 영향 없음).

### 1.2 자주 쓰는 유틸리티 클래스

| 클래스 | 용도 |
| --- | --- |
| `wiki-display` | 디스플레이 폰트 (Space Grotesk, 큰 제목 외에 사용) |
| `wiki-mono` | JetBrains Mono — 숫자/코드/라벨 |

| `wiki-h1`, `wiki-h2`, `wiki-h3` | 표제어 / 섹션 / 하위 섹션 (홀로 그라디언트 텍스트) |
| `wiki-pulse-dot` | 시안 LED, 펄스 애니메이션 |
| `wiki-panel`, `wiki-panel--inset` | HUD 카드 (코너 브래킷 자동 적용) |
| `wiki-divider` | 가운데가 진한 그라디언트 가로선 |

## 2. 새로 추가된 공통 컴포넌트

### 2.1 `WikiShell`

> [src/components/wiki/WikiShell.tsx](src/components/wiki/WikiShell.tsx)

```tsx
import { WikiShell, WikiPageHeader } from '@/components/wiki'

export default function MyWikiPage() {
  return (
    <WikiShell
      activeNav="recent"
      pageHeader={<WikiPageHeader … />}
      rightRail={<aside>…</aside>}
    >
      … 메인 컨텐츠 …
    </WikiShell>
  )
}
```

- 상단에는 펄스 도트 + LIVE 배지 + 검색 + 알림 + 운영(MOD) 배지 + 로그인 상태가 들어갑니다.
- 좌측은 “탐색 / 기여 / 도움말 / 다른 사이트” 4개 그룹 + 항목별 hover/active 글로우.
- `activeNav` 값: `'main' | 'recent' | 'category' | 'random' | 'search' | 'help' | 'mod' | 'workshop'`.
- `rightRail`은 xl 이상에서만 노출됩니다 (모바일에선 자동 숨김).

### 2.2 `WikiPageHeader`

```tsx
<WikiPageHeader
  title="이랑위키:대문"
  subtitle="여러분이 직접 가꾸는 작은 백과사전"
  hatnote={<>이 문서는 …</>}
  meta={[
    { label: '마지막 편집', value: '2026.05.02 13:19', icon: Clock },
    { label: '편집자', value: '관리자', icon: Users }
  ]}
  tabs={[
    { key: 'document', label: '문서', icon: Eye },
    { key: 'edit',     label: '편집', icon: Edit }
  ]}
  activeTab={activeTab}
  onTabChange={setActiveTab}
  actions={<>…헤더 우측 액션 버튼들…</>}
/>
```

이전 “정적 위키 헤더”에서 다음이 바뀌었습니다:
- 위에 `MISSION BRIEF` 시그널 라인 + 펄스 도트 자동 추가
- `wiki-h1`은 사이언→바이올렛→마젠타 그라디언트로 텍스트가 그려집니다
- `meta` 항목은 mono 폰트 + 어두운 카드 형태로 “HUD 상태 라인”처럼 표시됩니다
- 탭은 작전 모드 스타일로 활성 시 그라디언트 라인이 켜집니다

## 3. 문서 함수 패널 — 정리 결과

기존 `src/app/wiki/[...slug]/page.tsx`(1,500줄)에 통째로 들어가 있던
**문서 함수** 부분을 별도 파일로 분리했습니다.

### 3.1 새 위치

> [src/components/wiki/DocumentFunctions.tsx](src/components/wiki/DocumentFunctions.tsx)

내보내는 심볼:

```ts
export {
  DocumentFunctionsPanel,         // 우측 사이드바에 배치할 React 컴포넌트
  FUNCTION_DEFINITIONS,           // 등록된 함수 목록 (배열)
  sanitizeWikiText,               // 본문에서 마크업을 제거해 평문으로 만드는 헬퍼
}

export type {
  DocumentHeading,
  DocumentFunctionContext,
  DocumentFunctionResult,
  DocumentFunctionDefinition,
}
```

### 3.2 기본 함수 5종

| ID | 입력 필요 | 설명 |
| --- | --- | --- |
| `word-count` | ✕ | 단어 / 문단 / 본문 길이 (chars) 요약 |
| `reading-time` | ✕ | 320 wpm 기준 예상 읽기 시간 |
| `keyword-density` | ✓ | 키워드 등장 횟수 + 단어 대비 밀도 |
| `section-finder` | ✓ | 목차에서 키워드와 일치하는 섹션 + 클릭 스크롤 |
| `link-audit` ★ 신규 | ✕ | 내부 링크 / 외부 링크 / 각주 개수 |

### 3.3 함수 추가하는 방법

`FUNCTION_DEFINITIONS` 배열에 항목 한 개만 더하면 끝입니다.

```ts
{
  id: 'image-count',
  label: '이미지 개수',
  description: '문서에 삽입된 이미지의 개수를 셉니다.',
  icon: ImageIcon,                   // lucide-react 아이콘
  // requiresInput: true,            // 사용자 입력이 필요하면 추가
  // placeholder: '예: 키워드',
  run: (ctx, input) => {
    const count = (ctx.rawContent.match(/\[이미지:[^\]]+\]/g) || []).length
    return {
      summary: `이 문서에는 ${count.toLocaleString()}장의 이미지가 사용되었습니다.`,
      metrics: [{ label: '이미지', value: count.toLocaleString() }]
    }
  }
}
```

`run`에서 사용 가능한 컨텍스트:

```ts
{
  rawContent: string;   // 원본 위키 마크업
  plainText:  string;   // 마크업 제거 후 평문
  headings:   DocumentHeading[];   // tableOfContents 추출 결과
  wordCount:  number;
  title:      string;
}
```

리턴 형식:

```ts
{
  summary: string;                 // 한 줄 요약 (필수)
  metrics?: { label, value }[];    // 카드 형태로 표시
  matches?: { label, anchor? }[];  // 클릭하면 해당 anchor로 스크롤
}
```

`run`은 **동기 함수**입니다. 비동기 호출이 필요한 함수라면 별도 컴포넌트로 분리해 주세요.

## 4. 글을 작성할 때 — 빠른 문법 치트시트

### 4.1 기본 서식

| 결과 | 입력 |
| --- | --- |
| **굵게** | `'''굵게'''` 또는 `**굵게**` |
| *기울임* | `''기울임''` 또는 `*기울임*` |
| 밑줄 | `__밑줄__` |
| ~~취소선~~ | `~~취소선~~` |
| 윗첨자 / 아래첨자 | `^^위^^` / `,,아래,,` |
| 인라인 코드 | `` `code` `` (시안색 + 코드 박스) |

### 4.2 문서 구조

```
= 문서 제목 =
== 절 ==                (사이언 좌측 라인)
=== 소절 ===            (바이올렛 좌측 라인)
[[목차]]                (수동 목차 삽입)
```

### 4.3 링크

```
[[다른 문서]]            // 내부 링크 (시안 + dashed 밑줄)
[[다른 문서|보일 텍스트]] // 별칭
[표시할 텍스트](URL)     // 외부 링크
[*1] 또는 [*]           // 각주 (자동 번호)
```

### 4.4 분류

```
[[분류:게임]]
[[분류:게임|정렬키]]
```

저장 시 `extractCategoriesFromContent`가 자동으로 수집합니다.
별도 등록은 필요 없습니다.

### 4.5 표

```
||<bgcolor:#22D3EE> 사이언 셀 ||<rowspan=2> 머리글 ||
|| 일반 셀 || (자동) ||
```

색상 팔레트는 [src/lib/tableColors.ts](src/lib/tableColors.ts)에 정의되어 있습니다.
편집기 툴바 → **표 삽입** → **표 색상 설정**에서 GUI로도 만들 수 있어요.

### 4.6 템플릿 불러오기 (편집 탭)

```
/템플릿 인물
/template school
```

내부적으로 `/api/wiki/pages?title=템플릿:{분야}&namespace=template`을 호출해
`templates` 네임스페이스의 문서를 그대로 본문으로 가져옵니다.

새 템플릿이 필요하다면 `템플릿:OOOO` 제목으로 일반 문서를 만들면 됩니다.

### 4.7 이미지 삽입 규칙

`WikiEditor`의 이미지 버튼은 **커서 위치**를 보고 자동으로 자연스럽게 넣어 줍니다.

| 커서 위치 | 삽입 결과 |
| --- | --- |
| 표 셀 안 (`||` 줄) | `[이미지:URL]` 인라인 |
| `[[인포박스:…]]` 안 | `\| 이미지=URL` 필드 자동 갱신 |
| `{{인물정보상자 …}}` 안 | `\| 이미지=URL` 자동 추가 |
| `[[카드그리드:…]]` 안 | `items[0].image`에 자동 매핑 |
| 빈 줄 | 블록형 이미지 |
| 그 외 | 인라인 이미지 |

업로드 한도는 1MB이며 `/api/wiki/files/upload`로 멀티파트 업로드됩니다.

### 4.8 단축키

| 키 | 동작 |
| --- | --- |
| `Ctrl + B` | 굵게 |
| `Ctrl + I` | 기울임 |
| `Ctrl + S` | 저장 |
| `Ctrl + Enter` | 미리보기 토글 |
| `Tab` | 들여쓰기 (스페이스 2칸) |

## 5. 운영 동작은 그대로

리디자인은 **보이는 부분에만** 손을 댔고 API 동작은 변경하지 않았습니다.

| 동작 | 엔드포인트 | UI 위치 |
| --- | --- | --- |
| 페이지 조회 / 저장 / 생성 | `GET / POST / PUT /api/wiki/pages` | 문서 / 편집 탭 |
| 리비전 비교 / 되돌리기 | `/api/wiki/pages/revisions`, `/api/wiki/pages/revert` | 역사 탭 |
| 보호 설정 | `POST /api/wiki/pages/protect` | 우측 사이드바 (운영자 전용) |
| 감시 목록 | `POST /api/wiki/watchlist` | 헤더 우측 “감시” 버튼 |
| 신고 | `POST /api/wiki/mod/reports` | 헤더 우측 “신고” 버튼 |
| 검수 처리 | `GET / POST /api/wiki/mod` | `/wiki/mod` |
| 토론 | `GET / POST /api/wiki/discussions` | 토론 탭 (신규 노출) |
| CAPTCHA 응답 | `POST /api/wiki/pages/captcha` | 본문 하단 숨김 컨테이너 |

## 6. 토론 탭 신규 노출

기존에는 토론 데이터를 가져오기만 하고 UI에서 직접 보여주지 않았습니다.
2026 리디자인부터는 문서 페이지에 **정식 탭**으로 노출되며,
로그인한 사용자는 누구나 새로운 토론을 시작할 수 있습니다.

```
문서 / 편집 / 역사 / 토론  ── 작전 모드 탭
                          ├─ "토론 시작하기" 카드 (제목 + 본문)
                          └─ 토론 목록 카드
```

답글 / 닫기 등 추가 기능을 붙이려면 `/api/wiki/discussions`의 `PUT`을 사용하세요.

## 8. 자주 하는 실수 / 운영 팁

1. **새 위키 페이지에 `theme-surface`를 직접 붙이지 마세요.**

3. **본문에서 `prose-invert`를 쓰지 마세요.**
   위키 영역은 다크 테마로 고정이라 prose-invert가 다시 반전을 만들어 어색해집니다.

4. **새 문서 함수는 동기 함수만**.
   네트워크가 필요하면 호출자가 미리 데이터를 가져와서 `context`에 같이 넣거나,
   별도 컴포넌트로 분리해 주세요.
5. **`WikiShell`을 `<main>` 안에 넣지 마세요.**
   자체적으로 `<header>` + `<main>` + `<aside>` 레이아웃을 포함합니다.

## 9. 새로 들어온 HUD 강화 기능 (2026 mid)

### 9.1 절 제목 옆 `[편집]` 링크

위키피디아처럼, 본문 모든 헤딩(h1~h6) 위에 마우스를 올리면 우측에
`[편집]` 미니 버튼이 떠오릅니다. 누르면 자동으로 편집 탭으로 전환되며,
같은 헤딩이 있는 라인까지 텍스트에어리어가 스크롤되고 커서가 그 줄을
선택해 줍니다.

- 구현: `[src/app/wiki/[...slug]/page.tsx`의 `useEffect` (action: 'document' 탭 진입 시)
  → 렌더된 `<article>` 안 헤딩에 동적으로 버튼을 주입.
- 스타일: `.wiki-section-edit` (globals.css)
- 편집 탭으로 이동할 때 헤딩 텍스트를 `sessionStorage`에
  잠시 저장해뒀다가 텍스트에어리어를 그 라인으로 스크롤시킵니다.

### 9.2 빨간 링크 (`wiki-redlink`)

본문에 들어 있는 내부 링크 중 **아직 만들어지지 않은 문서**는 자동으로
마젠타 색으로 표기됩니다. 클릭하면 그대로 새 문서 만들기 화면으로 이동합니다.

- 신규 엔드포인트: [src/app/api/wiki/exists/route.ts](src/app/api/wiki/exists/route.ts)
  - `GET  /api/wiki/exists?titles=A,B,C` 또는
  - `POST /api/wiki/exists  { titles: [...] }` (최대 200개)
  - 응답: `{ success: true, exists: { "A": true, "B": false } }`
- 클라이언트: 문서 탭이 열리면 본문 내 모든 `a[href^="/wiki/"]`를 모아
  배치 호출하고, 존재하지 않는 링크에 `.wiki-redlink` 클래스를 붙입니다.
- 검색·분류 등 라우트성 링크는 자동으로 검사에서 제외됩니다.

### 9.3 페이지 이동 시 챔버(chrome) 유지 + 콘텐츠 페이드 스왑

좌측 내비/상단 바/우측 인포박스가 페이지 이동마다 다시 마운트되지 않고
**한 번만** 마운트되며, 메인 콘텐츠 영역만 `framer-motion`으로 부드럽게
페이드-스왑됩니다.

- 새 파일: [src/app/wiki/layout.tsx](src/app/wiki/layout.tsx)
- 새 컴포넌트: `WikiShellLayoutFrame` (in `WikiShell.tsx`)
- 작동 방식:
  1. 레이아웃이 한 번만 챔버를 그립니다.
  2. 각 페이지의 `<WikiShell>`은 layout 컨텍스트를 감지해
     `pageHeader / rightRail / activeNav`만 등록합니다 (이중 챔버 X).
  3. 메인 콘텐츠와 우측 인포박스는 `pathname`을 키로 한
     `<AnimatePresence mode="wait">`로 페이드-스왑됩니다.
- 페이지 코드는 변경 없음 — 기존처럼 `<WikiShell ...>`만 쓰면 됩니다.

### 9.4 인포박스 진행 상태 (`<WikiMeter>`)

```tsx
import { WikiMeter } from '@/components/wiki'

<WikiMeter
  label="WATCHERS"
  valueLabel="12명"
  value={12}
  max={20}
  tone="violet"   // cyan | violet | magenta | warning | danger | success
  size="sm"        // sm | md
/>
```

- 1px-라인 안에 그라디언트 채움 + 글로우 + 미세한 스캔 셰이더.
- 숫자는 JetBrains Mono로 표현됩니다.
- 대문 인포박스(활동 점수 / 도움말 열람), 문서 인포박스(문서 크기 / 감시자)
  에서 이미 사용 중입니다.

### 9.5 스캔 라인 오버레이

문서 페이지 헤더에서 **`스캔`** 버튼을 누르면 본문에 시안 톤의 스캔 라인이
주기적으로 흘러내립니다.

- CSS 클래스: `wiki-scanlines` (`.wiki-content`에 토글로 추가)
- 키프레임: `wiki-scan` (4.2s linear, 무한)
- 토글 버튼은 헤더 액션 영역(공유 / 감시 / 신고 / 스캔 / 뒤로) 사이에 위치.
